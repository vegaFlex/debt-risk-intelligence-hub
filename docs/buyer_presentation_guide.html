1. What the Product Is

Short version

Debt & Risk Intelligence Hub is a debt portfolio analytics and acquisition review platform. It helps teams import debtor packages, monitor collection performance, generate reports, and evaluate whether a package is worth buying.

What makes it strong

  • Operational dashboard
  • Collection KPI tracking
  • Management reporting
  • Acquisition valuation layer
  • Collections intelligence layer
  • Role-based access
  • API and BI-ready structure

2. The Main Business Value

Portfolio visibility Risk prioritization Collections monitoring Acquisition pricing Next-best-action planning Management reporting

The platform turns raw debtor package data into a structured decision workflow. Instead of working from disconnected spreadsheets, a team can monitor performance, compare packages, review pricing signals, and prioritize debtor actions in one place.

3. Best Demo Order

Step 1 - Start with the Dashboard Explain that this is the operational control center. Show KPI cards, filters, charts, and the debtor preview.
Step 2 - Show the Report Preview Position it as the management-facing output layer. Explain that it turns operations data into exportable reporting.
Step 3 - Open the Valuation Workspace Explain that this is the acquisition layer. Show how portfolios are ranked by attractiveness, recovery, ROI, and confidence.
Step 4 - Open a Valuation Preview Show recommendation, scenarios, benchmark context, ML baseline forecast, and saved run history.
Step 5 - Open the Comparison Desk Compare two or three packages side by side and explain how this supports buy-side review.
Step 6 - Open the Strategy Workspace Show how the product moves from analytics into action by ranking debtors by next-best action, channel, priority, and expected uplift.
Step 7 - Mention the Admin Layer Keep this short. Position it as the maintenance and governance layer, not the main selling point.

4. Suggested Talk Track

This platform is built to support both debt operations and acquisition review. On the operational side, it lets teams import debtor portfolios, validate them, track KPI performance, and generate management-ready reports. On the acquisition side, it adds a valuation workflow that estimates recovery, recommends a bid range, compares packages side by side, and supports benchmark-aware review. On the collections side, it adds a strategy layer that recommends the next best action for each debtor, prioritizes queue work, and compares collection strategies before a team commits to one approach. The goal is to move from scattered spreadsheet review into a structured analytics product that supports daily collections work, higher-level portfolio decisions, and action planning.

6. Questions You Should Expect

"Can this work with our files?"

Yes, the current version supports validated structured imports. A future extension can add a richer mapping and normalization layer for heterogeneous source files.

"Is the valuation ML-driven?"

The current version combines rule-based pricing, historical benchmark fallback, similarity logic, and an ML-ready baseline forecast scaffold. It is built to evolve into a fuller trained valuation model.

"Does it help after a package is bought?"

Yes. The strategy layer extends the product beyond valuation by recommending next-best actions, prioritizing collector queues, and comparing collections strategies.

"Can different teams use different views?"

Yes. The app already supports role-based access, including a public read-only visitor account and more capable manager/admin flows.

"Can this feed BI tools?"

Yes. The platform already exposes a clean API layer and structured outputs that are suitable for BI consumption and future integrations.
